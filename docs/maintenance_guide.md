# AstraNova eスポーツチーム公式サイト - 運用保守手順書

本ドキュメントは、AstraNova公式サイトのデプロイ後の保守管理、エラー時のトラブルシューティング、およびコンテンツ（選手情報、実績等）の更新手順についてまとめたものです。

---

## 1. 動作監視とログの確認

お問い合わせ機能のトラブルシューティングや稼働状況を確認するには、Vercelのコンソールログを利用します。

### 1.1. Vercel サーバーレス関数のログ確認手順
1. [Vercel Dashboard](https://vercel.com/dashboard) にログインし、本プロジェクトを選択します。
2. 上部メニューから **Logs** タブをクリックします。
3. `/api/contact` に対するリクエストが一覧表示されます。
4. 特定のアクセスをクリックすると、詳細な実行ログ（リクエストヘッダ、エラー出力、`console.log` で仕込んだデバッグ情報など）を確認できます。
   - `Turnstile verification failed` や `Resend API Error` などの具体的な不具合原因を特定可能です。

---

## 2. 認証キーの更新・ローテーション手順

セキュリティを維持するため、またはキーの漏洩が疑われる場合は、定期的に認証キーを更新します。

### 2.1. Resend API キーのローテーション
1. [Resend](https://resend.com) で新しいAPIキーを作成します。
2. 古いAPIキーは**まだ削除しないでください**。
3. Vercelの **Settings > Environment Variables** にアクセスし、`RESEND_API_KEY` の値を新しいキーに書き換えます。
4. 本番環境に反映させるため、Vercelダッシュボードの **Deployments** タブから最新のデプロイに対して `Redeploy` を実行します（または `vercel --prod` コマンドで再デプロイします）。
5. 送信テストが成功したことを確認後、Resendの管理画面から古いAPIキーを削除します。

### 2.2. Cloudflare Turnstile キーのローテーション
1. [Cloudflare](https://dash.cloudflare.com) の Turnstile 設定画面で新しいシークレットキーとサイトキーを取得します。
2. `vercel.json` や HTML に埋め込んでいるサイトキーを変更し、Vercelダッシュボードの環境変数 `TURNSTILE_SECRET_KEY` を変更します。
3. プロジェクトを再デプロイし、動作確認を行います。

## 3. よくあるエラーと対策 (トラブルシューティング)

### 3.1. お問い合わせ送信時に `500 Internal Server Error` になる
* **原因1**: Vercelの環境変数 `RESEND_API_KEY` または `TURNSTILE_SECRET_KEY` が正しく設定されていない。
  - **対策**: Vercelダッシュボードの **Environment Variables** に値が正確に登録されているか確認し、再デプロイを行ってください。
* **原因2**: Resendの無料枠（送信制限数）を超過している。
  - **対策**: Resendのダッシュボードで送信制限ステータスを確認してください。

### 3.2. お問い合わせ送信時に `403 Forbidden` になる
* **原因1**: Cloudflare Turnstile のシークレットキーが正しくない、または期限切れ。
  - **対策**: Cloudflareダッシュボードでキーを再取得し、環境変数に再設定してください。
* **原因2**: Turnstile トークンの取得に失敗している、または有効期限（約2分）が切れた状態で送信ボタンを押した。
  - **対策**: ページをリロードし、再度Turnstileチェックを完了させて速やかに送信してください。

## 4. コンテンツ・デザインの更新手順

本サイトは Vanilla HTML/CSS で構築されているため、コンテンツの追加やデザイン変更は以下のファイルをエディタで直接書き換えます。

* **チーム実績の追加**:
  - `index.html` 内の `<section id="achievements">` にあるタイムライン要素（`<li>` または `div`）の構造をコピーして最上部に追加します。
* **メンバーの追加・引退**:
  - `index.html` 内の `<section id="members">` にあるメンバーカード（`<div class="member-card">`）を追加または削除します。
  - 選手画像は、画像最適化のためWebP形式かつ `400px x 500px` 程度の一定の縦横比にトリミングしてから `public/assets/` に追加し、HTML側でパスを指定します。
* **テーマカラーの変更**:
  - `style.css` の `:root` セクションに定義されているカラーコード（例: `--accent-cyan`）を書き換えることで、サイト全体のアセントカラーを一斉に変更できます。
