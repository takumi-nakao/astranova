# AstraNova eスポーツチーム公式サイト - テスト仕様書

本ドキュメントは、AstraNova公式サイトが機能、デザイン、およびセキュリティ面で要件を満たしているか確認するためのテストケースと検証手順を定義したものです。

---

## 1. デザイン・マルチデバイス対応テスト

| テストID | テスト対象 | テスト手順・内容 | 期待される結果 | 判定 |
| :--- | :--- | :--- | :--- | :--- |
| DD-01 | レスポンシブ (PC) | Chrome DevToolsで `1200px` 以上の画面幅で確認。 | ヒーロー画像が全画面表示され、余白が美しく保たれ、ナビゲーションが横並びであること。 | |
| DD-02 | レスポンシブ (Tablet) | 画面幅を `768px ~ 1023px` に設定して確認。 | 各種カード（Members, Gallery）が2カラムまたは適正な配置になり、はみ出しがないこと。 | |
| DD-03 | レスポンシブ (Mobile) | 画面幅を `375px ~ 480px` に設定して確認。 | ナビゲーションがハンバーガーメニューに切り替わり、カードが1カラム表示に切り替わること。 | |
| DD-04 | ブラウザ互換性 | Chrome, Safari, Edge, Firefox でサイトを開く。 | 各種CSS（Grid, Flex, backdrop-filter）が正しく描画され、フォント崩れがないこと。 | |

## 2. UI/UX・アニメーション機能テスト

| テストID | テスト対象 | テスト手順・内容 | 期待される結果 | 判定 |
| :--- | :--- | :--- | :--- | :--- |
| UX-01 | ローディング画面 | ページをリロードする。 | 0%から100%へのカウントアップが表示され、100%後にロード画面がフェードアウトすること。 | |
| UX-02 | カスタムカーソル | マウスを画面内で動かす。 | シアン色のリングカーソルがマウスに遅れて滑らかに追従すること。 | |
| UX-03 | カーソルホバー | リンク (`a`) やボタン (`button`) にマウスを重ねる。 | リングカーソルが拡大し、背景色が変化（反転）するなどのホバーフィードバックがあること。 | |
| UX-04 | スクロール表示 | ページを下部へスクロールする。 | 各セクションの要素が、画面内に入ると順次滑らかに下から上へスライドインすること。 | |
| UX-05 | 画像モーダル | ギャラリーセクションの画像をクリックする。 | 画面全体がぼやけた背景で暗くなり、クリックした画像が大きく表示されること。 | |
| UX-06 | モーダル閉鎖 | モーダル表示中に「モーダル外をクリック」または「Escキー」を押す。 | モーダルが閉じ、背面スクロールのロックが解除されること。 | |
| UX-07 | ハンバーガーメニュー | モバイル表示でメニューアイコンをクリックし、リンクを選択する。 | メニューが全画面で滑らかにスライドインし、リンクをクリックすると目的の場所にスクロールして自動で閉じること。 | |
| UX-08 | 進行インジケーター | ページを上下にスクロールする。 | 画面最上部のシアンの進捗バーが、現在のスクロール位置に正確に連動して伸縮すること。 | |

## 3. お問い合わせフォーム・API検証テスト

| テストID | テスト対象 | テスト手順・内容 | 期待される結果 | 判定 |
| :--- | :--- | :--- | :--- | :--- |
| API-01 | フロントバリデーション (空送信) | すべての項目を空のまま「送信」ボタンを押す。 | 送信されず、各入力フィールドの下に「必須入力」のエラーメッセージが出ること。 | |
| API-02 | メール形式バリデーション | メールアドレス欄に `test@invalid` などの不正フォーマットを入力。 | 「有効なメールアドレスを入力してください。」のエラーが表示されること。 | |
| API-03 | サーバーサイド制限 (超過) | お名前、メール、件名、本文それぞれで上限文字数を超えるデータをインジェクション送信。 | サーバーから `400 Bad Request` が返却され、送信失敗メッセージが表示されること。 | |
| API-04 | Turnstile検証失敗 | Turnstileチェックを完了させずに強制的にPOSTを送信（デバッグ環境）。 | サーバーから `403 Forbidden` が返却され、「セキュリティチェックに失敗しました」と表示されること。 | |
| API-05 | 正常系テスト | すべての必須項目を入力し、Turnstileチェックをして送信ボタンを押す。 | ボタンが「送信中...」となり非活性化された後、送信成功のトースト通知（またはメッセージ）が表示され、入力値がクリアされること。管理者に通知メールが届くこと。 | |

## 4. セキュリティ・Lighthouse品質検証

| テストID | テスト対象 | テスト手順・内容 | 期待される結果 | 判定 |
| :--- | :--- | :--- | :--- | :--- |
| SEC-01 | セキュリティヘッダ確認 | デプロイ後、[Security Headers](https://securityheaders.com/) にてサイトドメインを検証。 | CSP, HSTS, X-Frame-Options等の存在が確認され、A評価以上が獲得できていること。 | |

| PERF-01 | Lighthouse検証 | Chrome DevToolsのLighthouseにて、「Performance」「Accessibility」「Best Practices」「SEO」を検証。 | 各指標が90以上（特にバニラJSによるパフォーマンスとSEOが最大化されていること）を達成。 | |
