# AstraNova eスポーツチーム公式サイト - 要件定義書

本ドキュメントは、プロeスポーツチーム「AstraNova（ネクシス）」の公式サイト制作における要件定義をまとめたものです。本サイトはWeb制作のポートフォリオとして機能し、訪問者（潜在的なクライアント、スポンサー企業、ファン）に対して高い制作スキルとブランド表現力をアピールすることを目的とします。

---

## 1. プロジェクト概要
* **システム名**: AstraNova eスポーツチーム公式サイト
* **対象チーム**: AstraNova (Apex Legends, VALORANT等のマルチゲーム部門を擁する仮想のプロeスポーツチーム)
* **開発のねらい**:
  - チームのブランディング強化（高級感、近未来感、ミニマルデザインによるプレミアム感の演出）
  - スポンサー獲得のための信頼性構築および窓口の設置
  - 新規所属メンバー・クリエイターの募集
  - 開発者自身のWeb制作技術（Vanilla JSによる高度なアニメーション、セキュリティ対策、サーバーレスAPI等）の実証

---

## 2. ターゲットユーザーと提供価値
1. **Webサイト制作依頼を検討するクライアント（最重要対象）**
   - **提供価値**: 「この開発者にハイクオリティなWebサイト制作を依頼したい」と思わせるUI/UX、滑らかなモーション、堅牢なセキュリティ設計の実例提示。
2. **スポンサー協賛を検討する企業**
   - **提供価値**: スポンサーロゴが美しく調和するクリーンなレイアウト、チームの実績や信頼性をクリアに示すブランド表現。
3. **チームのファンおよび一般ゲーマー**
   - **提供価値**: 所属選手や大会実績を直感的に把握できる洗練されたデザインと快適な操作性。

---

## 3. 機能要件

### 3.1. フロントエンド（画面表示・UI機能）
* **フルスクリーンヒーロー**: ファーストビューで圧倒的なインパクトを与えるビジュアルと大型キャッチコピー、CTAボタン、スクロール誘導。
* **チーム概要（About）**: チームのビジョン、活動内容、ミッションの掲載。
* **メンバー紹介（Members）**: 所属プレイヤー・ストリーマーのカード型表示。各SNS（X, YouTube, Twitch等）へのリンクおよびインタラクティブなホバー演出。
* **実績タイムライン（Achievements）**: 大会での主要実績・戦績を時系列で綺麗にレイアウト。
* **ギャラリー（Gallery）**: 大会やチーム活動のハイライト画像をグリッド表示。クリック時にフルスクリーンでプレビューするモーダルウィンドウ機能。
* **スポンサー枠（Sponsors）**: スポンサー企業のロゴ一覧（調和を乱さないモノクロ表示、ホバーでカラー化）とスポンサー募集用のCTA。
* **お問い合わせ（Contact）**: セキュアなお問い合わせフォーム（氏名、メールアドレス、件名、本文）およびCloudflare TurnstileによるBot保護。
* **共通UI/UX**:
  - スムーススクロール（アンカーリンク移動時）
  - IntersectionObserverによるスクロール出現フェードイン
  - フルスクリーンハンバーガーメニュー
  - ロード完了までのプログレスインジケーター（ローディング画面）
  - 微細なパララックススクロール効果
  - マウス追従型のインタラクティブカスタムカーソル
  - スクロール進行度インジケーター

### 3.2. バックエンド（API機能）
* **お問い合わせ送信API**:
  - `POST /api/contact` エンドポイントの実装。
  - 送信データのバリデーション（サーバーサイド）およびサニタイズ。
  - Cloudflare Turnstile トークンの検証。
  - **Resend API** を用いたGmail宛てのメール通知送信。
  - セキュアな環境変数管理（Resend APIキー、Turnstileシークレットキー）。

---

## 4. 非機能要件

### 4.1. デザイン・UX要件
* **テーマ**: ダークテーマ（高級感、近未来感、ミニマルデザイン、洗練された余白設計）。
* **禁止事項**: 安っぽいネオンカラーの多用、過剰で重いパーティクル演出、古典的で無駄に派手なゲーミングデザインの排除。
* **タイポグラフィ**: 視認性が高く、先進的な幾何学的フォントの採用。

### 4.2. セキュリティ要件
* **通信**: 全経路HTTPS必須（Vercel標準）。
* **レスポンスヘッダの保護**: CSP (Content-Security-Policy), HSTS, X-Frame-Options: DENY, X-Content-Type-Options: nosniff などのセキュリティヘッダを `vercel.json` で強制。
* **インラインスクリプト禁止**: HTML内に直接 `<script>` を記述せず、すべて外部ファイル（`script.js`）およびCSPのホワイトリストで運用。
* **Bot対策**: Cloudflare Turnstile によるフォームのスパム保護。

### 4.3. パフォーマンス要件
* **Core Web Vitals の最大化**:
  - Lighthouseスコア90以上を意識した軽量設計。
  - 画像の遅延ロード（`loading="lazy"`）とWebP等の最適化形式の採用。
  - 外部ライブラリ（jQuery, React等）を一切使わない純粋なバニラコードによるJS容量の最小化。

### 4.4. レスポンシブ対応
* PC、タブレット、スマートフォンの各画面幅に対する完全な最適化（ブレイクポイント: `768px`, `1024px`）。
